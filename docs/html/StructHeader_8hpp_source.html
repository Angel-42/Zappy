1/*
2** EPITECH PROJECT, 2024
3** Zappy
4** File description:
5** StructHeader.hpp
6**
7** This header defines core mathematical and data structures used for 3D model
8** representation and manipulation, particularly for OBJ file loading and rendering.
9**
10** Structures:
11** - Vec2: 2D vector with arithmetic, normalization, and dot product operations.
12** - Vec3: 3D vector with arithmetic, normalization, dot/cross product operations.
13** - Vec4: 4D vector, often used for homogeneous coordinates, with arithmetic and conversion to Vec3.
14** - Ray: Represents a 3D ray with origin and direction.
15** - Tri: Represents a triangle with 3 vertices and optional UV coordinates.
16** - Face: Stores indices for vertices, UVs, and normals for a polygonal face.
17** - Material: Stores material properties (colors, shininess, texture paths, etc.) for rendering.
18** - OBJModel: Aggregates all geometry and material data for a loaded OBJ model.
19**
20** Additional Features:
21** - Equality and three-way comparison operators for Vec2, Vec3, Vec4.
22** - std::hash specializations for Vec2, Vec3, Vec4 to allow their use in hash-based containers.
23**
24** Dependencies:
25** - Standard C++ headers: <iostream>, <fstream>, <sstream>, <string>, <vector>, <cmath>, <optional>, <compare>, <functional>
26**
27** Usage:
28** Include this header in any module that needs to manipulate 2D/3D/4D vectors,
29** handle OBJ model data, or work with materials in a rendering context.
30*/
