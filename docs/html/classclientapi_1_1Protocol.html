Detailed Description

Static utility class containing all Zappy network protocol constants and helper functions.

This class centralizes all protocol-related constants and provides utilities for:

  • Command validation and parsing
  • Message formatting according to Zappy specification
  • Time limit management for different commands
  • Protocol compliance checking

The Protocol class implements the exact specifications from the Zappy project requirements:

  • AI client commands with proper time limits
  • Standard response messages (ok, ko, dead, etc.)
  • Command buffer limits (max 10 commands)
  • Authentication flow (WELCOME -> team name -> game info)
Note
All members are static - this is a utility class, not meant to be instantiated

Maximum number of commands a client can buffer before server drops new ones.

From specification: "The client can send up to 10 requests in a row without any response from the server. Over 10, the server will no longer take them into account."
