Detailed Description

Manages automatic life consumption and food management for all players.

According to the PDF specification:

  • Players consume 1 life unit per time unit automatically
  • Players start with 10 food units = 1260 life units
  • Players die when life reaches 0 and they have no food
  • Food is automatically consumed when life gets low

This class integrates with the TimeManager to process life consumption every game tick and handles player death notifications.

◆ processLifeConsumption()

void clientapi::LifeManager::processLifeConsumption ( int  timeUnits)

Processes life consumption for all registered players.

Parameters
timeUnitsNumber of time units to advance

This method:

  1. Calls consumeLifeOverTime() for each alive player
  2. Handles automatic food consumption
  3. Detects player deaths and triggers callbacks
  4. Updates player states

Should be called every game tick by the main game loop.
